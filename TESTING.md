# Руководство по тестированию — Bike Rental App

## Подготовка

### Запуск
```
1. Бэкенд:   cd vite-project/backend && npm start
2. Фронтенд: cd vite-project && npm run dev
3. Браузер:  http://localhost:5173
```

### Доступ к БД (для проверки)
```
PGPASSWORD=1515 "C:\Program Files\PostgreSQL\14\bin\psql" -U postgres -d bikerental
```
Базовые команды:
```sql
-- Последние договоры
SELECT id, status, prepayment_amount, deposit_type, deposit_amount, total_price FROM rental_contracts ORDER BY id DESC LIMIT 5;

-- Позиции договора
SELECT id, status, price, paid_amount, discount_type, discount_percent FROM rental_items WHERE contract_id = <ID>;

-- Статус велосипеда
SELECT id, bike_number, condition_status FROM bikes WHERE id = <ID>;

-- Статус клиента
SELECT id, last_name, first_name, status, no_show_count FROM customers WHERE id = <ID>;
```

## Статусы договора

```
booked → active → completed
booked → cancelled
booked → no_show
active → overdue (вручную, устарело)
```

## Сценарии тестирования

---

### СЦЕНАРИЙ 1 — Простая бронь без предоплаты, оплата при получении

**Шаги:**
1. /rentals → "Создать бронь"
2. Найти клиента (2+ символа), выбрать
3. Добавить велосипед: указать тариф, даты начала/конца
4. Предоплата = 0, Залог = Без залога
5. Нажать "Создать бронь"

**Ожидаем:**
- Открылась карточка договора (RentalViewModal)
- Статус: "Забронирован"
- Кнопки: "Выдать велосипед" / "Не явился" / "Отменить"

**БД:**
```sql
SELECT status, prepayment_amount, deposit_type FROM rental_contracts WHERE id = <ID>;
-- status = 'booked', prepayment_amount = null или 0, deposit_type = 'none'
SELECT status, actual_start FROM rental_items WHERE contract_id = <ID>;
-- status = 'active', actual_start = null (ещё не выдан)
SELECT condition_status FROM bikes WHERE id = <bike_id>;
-- 'бронь'
```

---

### СЦЕНАРИЙ 2 — Активация брони (выдача велосипеда)

**Продолжение сценария 1.**

**Шаги:**
1. В карточке договора нажать "Выдать велосипед"
2. Указать выдавшего сотрудника
3. Нажать "Выдать"

**Ожидаем:**
- Статус изменился на "Активен"
- В блоке Оплата видно период брони
- Появился блок "Расчёт" с живой ценой
- Кнопка "Принять X ₴ и завершить" или "Завершить прокат"

**БД:**
```sql
SELECT status, actual_start FROM rental_contracts WHERE id = <ID>;
-- status = 'active', actual_start = NOW()
SELECT actual_start FROM rental_items WHERE contract_id = <ID>;
-- actual_start заполнен
SELECT condition_status FROM bikes WHERE id = <bike_id>;
-- 'в прокате'
```

---

### СЦЕНАРИЙ 3 — Завершение с оплатой в конце (без предоплаты)

**Продолжение сценария 2.**

**Шаги:**
1. В карточке активного договора видим блок "Расчёт"
2. Проверить что сумма пересчитывается в реальном времени (live-цена)
3. При необходимости заполнить "Принял" (сотрудник)
4. Нажать "Принять X ₴ и завершить"
5. В ConfirmModal нажать подтверждение

**Ожидаем:**
- Модалка закрылась
- В таблице договор в статусе "Завершён"
- При открытии завершённого договора: блок расчёта показывает Базовая / Скидки (если были) / Оплачено / Итого ✓ оплачено

**БД:**
```sql
SELECT status, total_price, actual_end FROM rental_contracts WHERE id = <ID>;
-- status = 'completed', total_price заполнен, actual_end = NOW()
SELECT status, paid_amount, actual_end FROM rental_items WHERE contract_id = <ID>;
-- status = 'returned', paid_amount = сумма позиции, actual_end заполнен
SELECT condition_status FROM bikes WHERE id = <bike_id>;
-- 'в наличии'
```

### СЦЕНАРИЙ 4 — Бронь с предоплатой

**Шаги:**
1. Создать бронь
2. В поле "Предоплата" ввести сумму (например 300 ₴)
3. Создать бронь → выдать велосипед

**Ожидаем при выдаче:**
- В блоке "Оплата" видна предоплата 300 ₴
- В блоке "Расчёт" строка "Предоплата −300 ₴"
- Кнопка показывает "Принять X ₴ и завершить" где X = цена − 300

**При завершении:**
- X = максимум(0, живая цена − 300)
- Если предоплата перекрывает всю сумму — кнопка "Завершить прокат" (без суммы)

**БД:**
```sql
SELECT prepayment_amount, total_price FROM rental_contracts WHERE id = <ID>;
-- prepayment_amount = 300
SELECT paid_amount FROM rental_items WHERE contract_id = <ID>;
-- paid_amount = полная цена позиции (не toPay), т.к. предоплата учитывается на уровне договора
```

### СЦЕНАРИЙ 5 — Бронь с залогом (денежный)

**Шаги:**
1. Создать бронь, выбрать Залог = "Денежный", сумма например 500 ₴
2. Создать и выдать

**Ожидаем:**
- В блоке "Залог" карточки договора: "Денежный 500 ₴" + "Не возвращён"
- После завершения: отметить залог возвращённым (если есть кнопка)

**БД:**
```sql
SELECT deposit_type, deposit_amount, deposit_returned, deposit_returned_at FROM rental_contracts WHERE id = <ID>;
-- deposit_type = 'money', deposit_amount = 500
```

---

### СЦЕНАРИЙ 6 — Залог документом

**Шаги:**
1. Создать бронь, Залог = "Документ", в поле ввести "Паспорт АА123456"

**БД:**
```sql
SELECT deposit_type, deposit_value FROM rental_contracts WHERE id = <ID>;
-- deposit_type = 'document', deposit_value = 'Паспорт АА123456'
```

### СЦЕНАРИЙ 7 — Частичный возврат (несколько велосипедов)

**Шаги:**
1. Создать активный прокат с 2 велосипедами
2. В карточке договора у первого велосипеда нажать ↩ (возврат)
3. Проверить что live-цена подтянулась в поле оплаты
4. Нажать "Принять оплату X ₴"
5. Второй велосипед остаётся активным

**Ожидаем:**
- Первая позиция: статус "Возвращён", серая цена = paid_amount (без ,00)
- Вторая позиция: всё ещё активна, показывает live-цену
- Блок Расчёт: "Оплачено ранее −X ₴" + живая цена второго

**При завершении второго:**
- paid_amount второго тоже сохраняется
- В завершённом договоре обе позиции показывают paid_amount

**БД:**
```sql
SELECT id, status, paid_amount FROM rental_items WHERE contract_id = <ID>;
-- Первая: status='returned', paid_amount = число
-- Вторая после завершения: status='returned', paid_amount = число
```

### СЦЕНАРИЙ 8 — Скидки

#### 8а — Скидка УБД (−20%)
1. Клиент с флагом "УБД" — при добавлении в модалку должен появиться алерт "🎖 УБД"
2. Скидка 20% должна применяться автоматически

#### 8б — Скидка Именинник (−50%)
1. Клиент у которого ДР в течение 7 дней — алерт "🎂 Именинник"
2. Скидка 50% на 1 велосипед

#### 8в — Групповая скидка (−10%)
1. Добавить 5+ велосипедов в один договор
2. Скидка 10% применяется автоматически на все позиции

#### Проверка скидки при возврате:
- В карточке позиции видна метка скидки (например "Группа −10%")
- Цена в поле оплаты уже пересчитана со скидкой

**БД:**
```sql
SELECT discount_type, discount_percent, paid_amount FROM rental_items WHERE contract_id = <ID>;
-- discount_type = 'veteran'/'birthday'/'group', discount_percent = 20/50/10
```

**БД:**
```sql

SELECT condition_status FROM bikes WHERE id = <bike_id>;
```

### СЦЕНАРИЙ 10 — Неявка клиента

**Шаги:**
1. Открыть бронь
2. Нажать "Не явился"
3. Подтвердить в ConfirmModal

**Ожидаем:**
- Договор статус "Не явился"
- У клиента счётчик no_show_count увеличился на 1
- Если no_show_count >= 2 — клиент автоматически получает статус "Без бронирования"

**БД:**
```sql
SELECT status FROM rental_contracts WHERE id = <ID>;
-- 'no_show'
SELECT no_show_count, status FROM customers WHERE id = <customer_id>;
-- no_show_count увеличился; если >= 2, status = 'no_booking'
```

## Проверка UI-элементов

### ConfirmModal (диалог подтверждения)
Должен появляться при:
- [ ] Завершение проката
- [ ] Отмена брони
- [ ] Неявка клиента
- [ ] Удаление договора (кнопка в таблице)
- [ ] Удаление клиента
- [ ] Удаление тарифа
- [ ] Удаление велосипеда

Кнопка "Отмена" — серая, нейтральная.
Кнопка подтверждения опасного действия — красная.

### Блок расчёта в карточке договора
| Статус договора | Блок виден? | Что показывает |
|---|---|---|
| Активен | Да | Live-цена + скидки + предоплата + toPay |
| Завершён | Да | Базовая / Скидки / Оплачено / Итого ✓ |
| Забронирован | Нет | — |
| Отменён | Нет | — |

### Карточки позиций (велосипедов)
| Статус позиции | Базовая цена | Финальная цена | Скидка |
|---|---|---|---|
| Активна | live-цена (или из брони) | после скидки | метка |
| Возвращена | пусто | paid_amount из БД | метка |

## Известные ограничения

- Удаление договора — только для статуса "Забронирован"
- Возврат предоплаты — поле `deposit_returned` только информационное, логики автовозврата нет
- Статус "Просрочен" — выставляется вручную, автоматики нет
